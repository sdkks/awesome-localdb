# AI Contribution Policy

This project welcomes contributions made with AI assistance, under clear and transparent conditions.

## Our Stance

This project itself is developed with AI assistance. We believe AI tools can accelerate development, improve code quality, and help more people contribute. We also believe that AI-assisted contributions must be reviewed and validated by a human before being submitted.

## Requirements for AI-Assisted Contributions

If you use AI tools (Claude, ChatGPT, Copilot, etc.) while preparing a contribution, you must:

1. **Disclose it.** In your pull request, check the AI disclosure box and briefly describe which tools you used and how (e.g., "Used Claude Code to generate the initial recipe structure, then manually reviewed and tested").

2. **Review it.** You are responsible for every line of code you submit. AI-generated code must be manually reviewed for correctness, security, idiomatic style, and adherence to this project's conventions.

3. **Test it.** AI-generated code must pass all local tests before submission. Run the language-specific test suite and `bash scripts/build-index.sh`. Untested AI output is not acceptable.

4. **Understand it.** Do not submit code you cannot explain. If asked during review, you should be able to describe what the code does and why you chose that approach.

## What We Do Not Accept

- Fully automated PRs with no human review (bot-generated contributions)
- AI-generated code that copies substantial portions of copyrighted works without attribution
- Contributions where the submitter cannot explain the changes

## Attribution

When using AI tools, follow the license terms of those tools. This project's MIT license applies to all accepted contributions. You retain copyright of your original work; the AI tool provider's terms govern your use of their output.

## Questions

If you are unsure whether your use of AI tools complies with this policy, open an issue and ask before submitting a PR.
